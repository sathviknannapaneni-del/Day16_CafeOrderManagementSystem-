# Day16_CafeOrderManagementSystem-

This C program simulates a simple Café Order Management System that helps manage customer orders and daily sales using 1D arrays, loops, and conditionals.

* Main Features

Displays a fixed menu of 5 items with prices.

Accepts orders for up to 10 customers.

For each customer:

Prompts for the number of items.

Takes item number and quantity for each.

Calculates and displays the total bill.

Tracks total quantities sold for each menu item.

After all customers are processed, it displays:

Total revenue (sum of all customer bills)

Total items sold

Most ordered item

Least ordered item

* Key Concepts Used

1D Arrays → To store prices, item names, and quantities sold.

Loops (for) → To handle multiple customers and multiple items per customer.

Conditional Statements (if–else) → To validate item numbers and find most/least ordered items.

Variables → To accumulate totals for revenue and item counts.

* Working Process

Initialize arrays and totals to zero.

Display the menu to the user.

Input the number of customers.

For each customer:

Input number of items.

For each item, read the item number and quantity.

Update customer total and café totals.

After all orders:

Loop through qtySold[] to find the maximum and minimum ordered items.

Print the final café summary.

* Final Output

The program neatly displays:

Each customer’s total bill

Café summary including:

Total Revenue

Total Items Sold

Most Ordered Item

Least Ordered It
